# SK클린텍 코딩 규칙

프로그래밍 시 반드시 준수할 규칙입니다.

---

## 1. 코딩 전 필수

| 규칙 | 설명 |
|------|------|
| **가이드 확인** | `PROJECT_GUIDE.md`, `ARCHITECTURE.md`를 먼저 읽고 작업 |
| **전체 룰 파악** | 수정 범위를 명확히 한 뒤 코딩 시작 |
| **영향 범위** | 변경이 다른 기능에 미치는 영향을 항상 고려 |

---

## 2. 모듈화 (절대 원칙)

- **한 서버나 한 파일에 코드를 몰아 넣지 않는다**
- 기능별로 독립된 모듈/폴더로 분리
- 각 모듈은 자신의 도메인만 담당
- 수정 시 해당 모듈만 수정해도 되도록 설계
- **시스템이 가볍게 동작**하도록 유지

---

## 3. 수정 편의성

- 코딩 수정이 **매우 편해야** 함
- 한 곳을 고치면 다른 곳이 깨지지 않도록
- 공통 타입/인터페이스는 별도 파일로 분리
- 설정값은 환경변수 또는 설정 파일로 분리

---

## 4. DB 및 외부 서버

- **외부 웹서버** 사용 전제
- **DB 테이블이 외부 DB 서버에 정확히 반영**되도록 설계
- Prisma 마이그레이션으로 스키마 관리
- `DATABASE_URL` 등은 환경변수로만 관리
- 로컬 ↔ 외부 전환 시 설정만 변경하면 되도록

---

## 5. 다른 기능 보호

- **코딩 시 다른 기능에 문제가 생기면 안 됨**
- 공통 인터페이스 변경 시 사용처 전체 검토
- 기존 API·컴포넌트 시그니처는 호환성 유지
- 새 기능 추가 시 기존 코드 최소 수정

---

## 6. 디자인

- **쿠팡 스타일**: 심플하고 간결
- 불필요한 장식·애니메이션 최소화
- 흰색/회색 계열, 넉넉한 여백
- 버튼·카드·리스트는 단순한 형태

---

## 7. 메인 대시보드

- DB 접수 화면이 **메인 대시보드** 역할
- 스프레드시트 헤더 기반 입력 항목 (방·화·베, 통화시도, 주소 등)
- 우체국 주소 검색 연동
- 접수 → 분배 → 팀장 전달 흐름 지원
