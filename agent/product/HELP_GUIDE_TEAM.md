# 팀장 사용 설명서 (청소비서)

> 이 문서는 입주·이사 청소 현장을 담당하는 팀장이 사용하는 화면들을 설명합니다.  
> 각 화면의 기능, 주요 버튼, 사용 방법을 단계별로 정리했습니다.

---

## 1. 오늘 일정 (대시보드) — `/team/dashboard`

**화면 목표**  
팀장이 앱에 로그인하면 가장 먼저 보는 화면입니다.  
**오늘 배정된 모든 현장을 한눈에 확인**하고, 고객 특이사항과 팀원 배치를 빠르게 파악합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **헤더** | 오늘 날짜, 배정된 현장 건수, 처리 필요 사항 |
| **Happy Call 통계** | 미처리 고객 만족도 조사(Happy Call) 건수 표시 |
| **현장 카드 목록** | 각 현장의 고객명, 주소, 방/욕실/베란다 수, 특이사항 |
| **팀원 정보** | 함께 배치된 팀원 목록 |
| **상태 배지** | 현장 진행 상태 (예정, 시작, 완료, 미출동 등) |

**사용 방법**

1. 로그인하면 자동으로 이 화면이 표시됩니다.
2. 오늘 배정된 현장을 **위에서 아래로** 확인합니다.
3. 각 현장 카드를 **클릭**하면 상세 정보 모달이 열립니다.
   - 고객 전화번호, 상세 주소, 특수청소 옵션, 선호 시간대 등 전체 정보 확인
   - "청소 시작", "청소 완료" 등의 상태 변경 버튼

**FAQ**

**Q: 오늘 배정된 현장이 너무 많을 때는?**  
A: 위에서부터 순서대로 처리하세요. 각 현장의 "선호 시간대"를 참고해 순서를 조정할 수 있습니다.

**Q: Happy Call 통계는 뭐예요?**  
A: 청소 완료 후 고객에게 전화로 만족도를 조사하는 건입니다. "미처리" 수가 뜨면 우선 처리하세요.

**필요한 스크린샷**  
✅ 팀장 대시보드 메인 화면  
- 오늘 날짜
- Happy Call 통계 박스
- 3~4개 현장 카드 (다양한 상태 포함)

## 2. 배정 목록 — `/team/assignments`

**화면 목표**  
팀장에게 배정된 **모든 현장(과거, 현재, 미래)을 리스트로 검색·필터링**합니다.  
특정 기간이나 상태별로 현장을 찾을 때 사용합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **날짜 필터** | 탭: 오늘, 전체, 월간, 날짜 선택 |
| **정렬 기준** | 배정일시, 생성일, 선호 날짜로 정렬 가능 |
| **현장 테이블** | 날짜, 주소(시구), 방 정보, 팀원, 상태, 배정된 시간 |
| **페이지네이션** | 페이지 번호, 페이지 크기 선택 (20/50/100) |
| **검사 진행도 배지** | 현장검수 완료 진도 표시 |

**사용 방법**

1. 상단 탭에서 원하는 기간을 선택합니다.
   - **오늘**: 오늘 배정된 현장만
   - **전체**: 모든 현장
   - **월간**: 선택한 연도/월의 현장
   - **날짜**: 특정 날짜 입력

2. 테이블을 **위아래로 스크롤**해 현장을 둘러봅니다.

3. 특정 현장을 **클릭**하면 상세 정보 모달이 열립니다.

4. 필요시 **페이지 번호**를 이동하거나 **페이지 크기**를 변경합니다.

**FAQ**

**Q: "배정일시"는 뭐고, "선호 날짜"는 뭔가요?**  
A: 배정일시는 관리자가 현장을 팀장에게 배정한 시각입니다. 선호 날짜는 고객이 요청한 시공 예정일입니다.

**Q: 현장검수 배지는 뭐예요?**  
A: 팀장이 현장에서 작성해야 하는 체크리스트입니다. 진행도를 시각적으로 보여줍니다.

**필요한 스크린샷**  
✅ 배정 목록 테이블 전체 뷰  
- 여러 현장이 나열된 테이블
- 상단 날짜 필터 탭 (오늘/전체/월간/날짜)
- 페이지네이션 (페이지 1/3 등)

## 3. 스케줄 달력 — `/team/schedule`

**화면 목표**  
팀장의 배정 일정을 **월간 달력 형태**로 시각적으로 확인합니다.  
어느 날에 현장이 많은지, 휴무일은 언제인지 한눈에 파악합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **연도/월 선택** | 드롭다운으로 과거/미래 연도, 월을 선택 |
| **달력 그리드** | 일(Sun)~토(Sat) 배치, 각 날짜에 현장 건수 표시 |
| **일자 셀** | 현장 수 (0/1/2/3+), 색상으로 구분 |
| **Happy Call 통계** | 현재 월의 미처리/처리 대기 건수 |

**사용 방법**

1. 상단에서 **연도와 월**을 선택합니다.

2. 달력의 각 날짜를 확인합니다.
   - 숫자가 표시되면 그 날 배정된 현장 건수입니다.

3. 특정 날짜를 **클릭**하면 그 날의 현장 목록이 나타납니다.

4. 현장 카드를 **클릭**해 상세 정보를 확인합니다.

**FAQ**

**Q: 색상이 다르게 표시되는데 뭔가요?**  
A: 현장이 없으면 흰색, 1~2건이면 연한 색, 3건 이상이면 진한 색으로 표시됩니다.

**Q: 휴무 신청한 날은 어떻게 보이나요?**  
A: 달력에 "휴무" 배지가 표시되고, "휴무 신청" 탭에서 상세히 확인할 수 있습니다.

**필요한 스크린샷**  
✅ 월간 달력 뷰  
- 연도/월 선택 드롭다운
- 달력 그리드 (일주일 단위)
- 현장 수가 다양하게 표시된 여러 날짜

## 4. C/S (고객 문의) — `/team/cs`

**화면 목표**  
청소 완료 후 고객이 신청한 **AS(재시공), 재방문, 환불 요청** 등을 관리합니다.  
각 건의 상태를 추적하고 메모를 남깁니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **상태 탭** | 대기중, 처리중, 완료 (선택 가능) |
| **C/S 항목 목록** | 고객명, 현장 주소, 요청 사항, 신청 시간 |
| **상태 배지** | 현재 처리 상태 색상 표시 |
| **메모 입력창** | 처리 과정이나 결과를 기록 |
| **완료 버튼** | C/S 처리 완료 시 클릭 |

**사용 방법**

1. 상단 **상태 탭**을 선택합니다.
   - **대기중**: 아직 처리하지 않은 요청
   - **처리중**: 현재 진행 중인 요청
   - **완료**: 이미 처리된 요청

2. 각 항목을 **클릭**하면 상세 정보가 펼쳐집니다.
   - 고객 메시지 전체 내용
   - 현장 주소와 시공 날짜
   - 기존 메모 이력

3. 메모를 **입력**해 처리 과정을 기록합니다.

4. 요청을 처리했으면 **"완료" 버튼**을 클릭합니다.

**FAQ**

**Q: AS와 환불 요청은 어떻게 다르나요?**  
A: AS는 재시공(다시 청소)을 요청하는 것이고, 환불은 금액 돌려달라는 것입니다. 관리자와 상담 후 처리하세요.

**Q: 메모를 여러 번 추가할 수 있나요?**  
A: 예, 새로운 메모를 추가할 때마다 이력이 남습니다.

**필요한 스크린샷**  
✅ C/S 목록 화면  
- 상단 상태 탭 (대기중/처리중/완료)
- 여러 고객 문의 항목 리스트
- 하나의 항목 상세 정보 (메모 입력창 포함)

## 5. 전자계약 — `/team/e-contracts`

**화면 목표**  
팀장이 서명해야 할 **계약서(협력사 계약, 보수 계약 등)를 관리**합니다.  
계약서를 확인하고 전자서명(사인패드)으로 서명합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **날짜 필터** | 탭: 오늘, 전체, 월간, 날짜 선택 |
| **계약 목록** | 계약 이름, 발송일, 만료일, 서명 상태 |
| **상태 배지** | 미발송, 대기중, 서명필요, 서명완료, 만료, 폐기 |
| **"서명하기" 버튼** | 서명이 필요한 계약에만 활성화 |
| **페이지네이션** | 여러 계약이 있을 때 페이지 이동 |

**사용 방법**

1. 상단 **날짜 필터**에서 기간을 선택합니다.

2. 계약 항목을 **스크롤**해 현재 필요한 계약을 찾습니다.

3. 상태가 **"서명필요"** 또는 **"대기중"** 계약을 찾습니다.

4. **"서명하기" 버튼**을 클릭합니다.

5. 새 페이지에서 계약서 내용을 확인한 후 **사인패드에 사인**합니다.

6. **"서명 제출" 버튼**을 클릭하면 완료됩니다.

**FAQ**

**Q: 서명이 필요한데 버튼이 없어요.**  
A: 계약서가 폐기되었거나 만료되었을 수 있습니다. 관리자에게 확인하세요.

**Q: 실수로 잘못 서명했어요. 다시 할 수 있나요?**  
A: 관리자만 재발송 가능합니다. 관리자에게 문의하세요.

**필요한 스크린샷**  
✅ 전자계약 목록  
- 여러 계약 항목 (상태 다양: 완료, 대기중, 만료 등)
- 날짜 필터 탭
- "서명하기" 버튼이 있는 항목

✅ 계약 서명 페이지  
- 계약서 내용 (PDF 프리뷰)
- 사인패드 영역
- 서명 제출 버튼

## 6. 외부 정산 — `/team/settlement`

**화면 목표**  
팀장 본인이 **현장에서 벌어들인 수당, 정산금, 공제 내역을 확인**합니다.  
월별로 얼마를 받았는지, 공제는 무엇인지 상세히 확인합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **월 선택** | 연도/월 드롭다운 |
| **요약 탭** | 월간 총 수당, 공제, 실수령액 |
| **상세 탭** | 건별 정산 내역 (현장명, 금액, 상태) |
| **정산 항목** | 완료된 현장, 공제 내역, 환급 건 등 |
| **실수령액** | 최종 입금될 금액 강조 표시 |

**사용 방법**

1. 상단 **월 선택 드롭다운**에서 확인할 월을 선택합니다.

2. **요약 탭**을 보면:
   - 월간 총 수당
   - 공제 항목별 금액
   - 실수령액(최종 입금 예정액)

3. **상세 탭**을 클릭하면:
   - 각 현장별로 얼마를 받았는지 확인
   - 공제 사유 확인
   - 상태 (완료, 대기 등)

**FAQ**

**Q: "공제"가 뭐예요? 왜 돈이 깎여요?**  
A: 회사와의 협약에 따라 공동구매 비용, 보험료, 교육비 등이 공제될 수 있습니다. 상세는 관리자에게 문의하세요.

**Q: 이번 달 정산이 아직 안 나왔어요.**  
A: 월말 정산 마감일 이후에 나타납니다. 보통 월 말일 이후 5~10일 내에 최종 확인 가능합니다.

**필요한 스크린샷**  
✅ 외부 정산 - 요약 탭  
- 월 선택 드롭다운
- 총 수당, 공제액, 실수령액 (큰 숫자 강조)
- 요약/상세 탭 (요약 선택 상태)

✅ 외부 정산 - 상세 탭  
- 여러 현장의 정산 내역 테이블
- 현장명, 금액, 상태 컬럼

## 7. 휴무 신청 — `/team/dayoffs`

**화면 목표**  
팀장이 **휴무(현장 불가 날짜)를 신청**합니다.  
마음이 바뀌면 신청을 취소할 수 있고, 관리자의 승인 현황을 확인합니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **연도/월 선택** | 달력 조회 기간 선택 |
| **달력 그리드** | 일주일 단위, 각 날짜 클릭 가능 |
| **휴무 날짜** | 신청한 휴무일에 "휴무" 배지 표시 |
| **신청한 휴무 목록** | 날짜별로 신청한 휴무 일정과 상태 |
| **상태 배지** | 승인대기, 승인, 거절 |

**사용 방법**

1. 상단 **연도/월**을 선택합니다.

2. 달력에서 휴무 신청할 날짜를 **클릭**합니다.
   - 모달이 열려 확인을 요청합니다.
   - "추가" 버튼을 클릭하면 신청됩니다.

3. 이미 신청한 휴무는 달력에 **"휴무" 배지**로 표시됩니다.

4. 하단 **"신청한 휴무 목록"**에서:
   - 신청한 모든 휴무 확인
   - "승인대기" 상태면 아직 관리자 승인 안 됨
   - "거절" 상태면 관리자가 거부함

5. 신청을 **취소하려면** 목록에서 해당 휴무를 찾아 **"취소" 버튼**을 클릭합니다.

**FAQ**

**Q: 휴무 신청을 했는데 반영이 안 돼요.**  
A: 관리자의 승인 대기 중입니다. 상태 배지를 확인하세요. 승인되면 자동 반영됩니다.

**Q: 한 번에 여러 날짜를 휴무 신청할 수 있나요?**  
A: 날짜를 하나씩 클릭해 신청해야 합니다. 한 번에 여러 날짜 선택은 지원되지 않습니다.

**필요한 스크린샷**  
✅ 휴무 신청 달력  
- 연도/월 선택
- 달력 그리드 (일주일 단위)
- 휴무 배지가 표시된 날짜들
- 하단 "신청한 휴무 목록" (상태: 승인대기/승인/거절)

## 8. 메시지 — `/team/messages`

**화면 목표**  
팀장이 **관리자와 1:1 메시지로 소통**합니다.  
질문, 보고, 요청 사항을 실시간으로 주고받습니다.

**주요 구성 요소**

| 요소 | 설명 |
|------|------|
| **메시지 목록** | 시간 순서로 메시지 표시 (위: 오래된 것, 아래: 최신) |
| **팀장 메시지** | 오른쪽 정렬, 색상 구분 |
| **관리자 메시지** | 왼쪽 정렬, 색상 구분 |
| **읽음 표시** | 관리자가 메시지를 읽었는지 확인 |
| **입력창** | 메시지 작성 및 전송 버튼 |

**사용 방법**

1. 메시지 목록을 **위에서 아래로 스크롤**해 대화 내용을 확인합니다.

2. 화면 아래 **입력창에 메시지를 입력**합니다.

3. **"전송" 버튼** 또는 엔터키를 눌러 메시지를 보냅니다.

4. 메시지가 관리자에게 전달되면 목록에 나타나고, 관리자가 읽으면 **"읽음" 표시**가 됩니다.

5. 관리자의 답변을 기다립니다. (실시간 알림 수신)

**FAQ**

**Q: 관리자가 메시지를 안 읽은 것 같은데?**  
A: "읽음" 표시가 없으면 아직 읽지 않은 것입니다. 시간을 두고 다시 확인하세요. 긴급한 경우 관리자에게 전화하세요.

**Q: 이전 대화를 다시 볼 수 있나요?**  
A: 화면을 위로 스크롤하면 과거 메시지를 모두 확인할 수 있습니다.

**필요한 스크린샷**  
✅ 메시지 채팅 화면  
- 팀장과 관리자의 메시지 교환
- 여러 메시지 (기간 포함)
- 읽음/읽지 않음 표시
- 하단 입력창 및 전송 버튼

## 추가 화면: 현장검수

팀장이 배정받은 현장에 가서 직접 작성해야 하는 화면들입니다.

### 현장검수 — `/team/inspection/:inquiryId`
- **목적**: 청소 후 현장 상태를 체크리스트로 검사
- **포함 내용**: 방/욕실/베란다별 항목 확인, 특이사항 메모, 검사 완료 제출

### 청소 전 사진 — `/team/pre-clean-photo`
- **목적**: 청소 시작 전 현장 사진 촬영
- **포함 내용**: 각 공간별 사진 업로드, 고객에게 "Before" 사진으로 전달됨

### 청소 후 사진 — `/team/post-clean-photo`
- **목적**: 청소 완료 후 현장 사진 촬영
- **포함 내용**: 각 공간별 사진 업로드, 고객에게 "After" 사진으로 전달됨

## 사용 팁

1. **모바일 최적화**: 팀장 화면은 모두 **스마트폰(폰, 태블릿)**에서 사용하도록 설계됐습니다.
2. **네트워크**: 사진을 올리거나 데이터를 주고받을 때는 **Wi-Fi나 4G/5G 네트워크가 필요**합니다.
3. **로그아웃**: 다른 팀장이 같은 기기를 쓸 때는 꼭 **로그아웃**하세요. 개인 정보가 보입니다.
4. **지연**: 배정된 현장이 앱에 나타나는 데 **몇 분 걸릴 수 있습니다**. 새로고침(앱을 껐다 켜기) 해보세요.
