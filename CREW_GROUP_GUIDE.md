# 크루 그룹(현장 팀원 공유 로그인) 운영 가이드

이 문서는 **관리자**가 시스템에서 크루 그룹을 만들·설정하는 방법과, **현장 팀원**이 크루 메뉴(`/crew`)를 쓰는 방법을 단계별로 정리합니다.  
UI에 한국어·태국어가 함께 나오는 화면은 두 언어 모두 같은 기능을 가리킵니다.

---

## 1. 이 기능이 하는 일 (한 줄 요약)

- **하나의 로그인 ID**로 여러 현장 팀원이 **같은 화면**을 봅니다. (개인 계정이 아닌 **그룹 공유 계정**)
- **예약일(접수의 현장 예정일)** 기준으로, 접수에 적힌 **현장 인원 메모**와 맞는 팀원에게 일감이 붙어 **현장 일정**에 표시됩니다.
- **그룹장**을 지정하면, 그 사람만 **일자 명단** 저장·**설정**(표시명·연락처) 같은 편집을 할 수 있습니다. 그룹장이 **한 명도 없으면** 모두 **조회 전용**에 가깝게 동작합니다.

---

## 2. 관리자 가이드

관리자는 **일반 관리자 계정**으로 로그인한 뒤, **팀원 관리** 화면에서 크루 그룹을 다룹니다.

### 2.1 들어가는 경로

1. 관리자 로그인
2. 메뉴에서 **팀원 관리** (또는 동일 기능의 관리 화면)로 이동
3. 상단 근처 **「팀 크루 그룹 (공유 로그인)」** 섹션을 찾습니다.

### 2.2 사전 준비: 전사 팀원 풀

- 크루 그룹 **멤버**로 넣을 사람은 먼저 같은 화면 아래 **「팀원 목록」**에 등록되어 있어야 합니다.
- 여기서 등록한 **이름·연락처·사용 여부(활성/비활성)** 가 크루 화면과 연결됩니다.
- **표시용 태국어 이름(`nameTh`)** 은 관리자가 팀원 정보에서 넣을 수 있고, 그룹장은 크루 **설정** 화면에서도 보조 표시명을 수정할 수 있습니다(아래 3.5 참고).

### 2.3 새 크루 그룹 만들기

1. **「새 그룹」** 클릭
2. 다음 항목을 입력합니다.

| 항목 | 설명 |
|------|------|
| **그룹 이름** | 내부 구분용 이름 (예: `태국 A팀`). 로그인 ID와 별개입니다. |
| **로그인 ID** | 현장에서 쓸 **공유 아이디**. 영문·숫자 및 `@ . _ -` 만 사용, **3~64자**. 시스템 전체에서 **중복 불가**. |
| **공유 로그인 비밀번호** | 크루 로그인 시 사용. **4자 이상**. |
| **연락처(선택)** | 그룹 대표 번호 등. |
| **집계·일자 명단 모드** | 켜면(`useDailyRosterOnly`): 스케줄·접수 쪽에서 **그날 명단에 올라온 사람만** 현장 가용으로 취급합니다. 끄면: 그룹에 속한 **활성 팀원 전원**이 기본 후보가 됩니다. |
| **설정용 비밀번호(선택)** | **집계·일자 명단 모드가 켜진 그룹**에서, 그룹장이 **일자 명단을 저장**할 때 추가로 요구할 수 있는 **2차 비밀번호**. 공유 로그인 비밀번호와 **다릅니다**. 비워 두면 명단 저장 시 이 확인 없음. |
| **관리자 비밀번호** | 지금 로그인한 **관리자 본인 계정 비밀번호** (생성·삭제·민감 변경 시 확인용). |

3. 저장 후, 표에 **로그인 ID**가 보이면 현장에 그 ID로 안내하면 됩니다.

**제한:** 시스템에 크루 그룹은 **최대 30개**까지 만들 수 있습니다.

### 2.4 그룹 편집 (기존 그룹)

**「편집」** 으로 모달을 열면 다음을 바꿀 수 있습니다.

- 그룹명, 연락처, 로그인 ID, **집계 모드**, **사용/중지**
- **공유 로그인 비밀번호** 변경(새 비밀번호 입력 시)
- **설정용 비밀번호** 설정·변경·해제(관리자 비밀번호 + 폼 규칙 준수)
- **멤버 추가/제거**: 목록은 **전사 팀원 풀**에서만 선택
- **그룹장 지정**: 멤버 중 **한 명**을 그룹장으로 지정 가능. 그룹장이 있어야 크루 쪽에서 **일자 명단 저장**·**설정** 메뉴가 활성화됩니다.

**중지(`isActive: false`)** 한 그룹은 크루 로그인 시 **계정을 찾을 수 없거나 비활성** 으로 거절됩니다.

### 2.5 그룹 삭제

1. **「삭제」** 클릭
2. **관리자 본인 비밀번호** 입력 후 확인  
   → 데이터 정책에 따라 그룹·명단 등이 함께 정리됩니다. 운영 중인 그룹은 신중히 삭제하세요.

### 2.6 접수(현장 인원 메모)와 이름 매칭 — 왜 일정/홈 집계에 안 보일 수 있나

현장 일정·홈 **이달 접수 건수**는 모두 같은 규칙을 씁니다.

- 접수에 **현장 인원(크루 메모)** 필드에 이름을 적습니다.
- 메모는 쉼표·`·`·`/`·`|` 등으로 **여러 이름**을 나눌 수 있습니다.
- 그룹 멤버의 **한글 이름** 또는 **보조 표시명(태국어 등, `nameTh`)** 과 **완전히 동일하게** 맞아야 해당 멤버에게 일이 붙습니다. (앞뒤 공백은 무시)
- 접수 상태가 **취소·보류**이면 집계·일정 후보에서 빠집니다.

**운영 팁:** 태국 현장 스태프 이름은 메모·화면 모두에서 헷갈리지 않게 **한글 이름 + 태국어 표시명**을 맞춰 두는 것이 좋습니다.

## 3. 팀원(크루) 가이드 — `/crew` 메뉴 사용법

### 3.1 로그인 방법

1. 브라우저에서 서비스 **로그인 페이지**로 갑니다.
2. **「크루 공유 계정으로 로그인」** 을 체크합니다.
3. 관리자가 알려준 **그룹 로그인 ID**와 **공유 비밀번호**를 입력합니다.
4. 로그인에 성공하면 **크루 전용 화면**(`/crew`)으로 이동합니다.

**참고:** 세션이 끊기면 다시 로그인해야 합니다. 로그인 직후에는 **이전에 보던 크루 주소**로 돌아가도록 설계되어 있습니다.

### 3.2 화면 구성 (상단 메뉴)

| 메뉴 | 용도 |
|------|------|
| **홈** | 이번 달(또는 선택한 달) **멤버별 접수 건수** 막대 그래프. **‹ ›** 로 달 이동. |
| **일정** (현장 일정) | 날짜를 고르고, 그날 **팀원별 배정·시간·차량(팀장 정보)** 를 표 형태로 확인. |
| **일자 명단** | 달력에서 날짜를 골라 **그날 일할 수 있는 사람** 목록을 봅니다. **그룹장**은 여기서 명단을 편집·저장할 수 있습니다. |
| **설정** | **그룹장에게만** 보입니다. 멤버 **보조 표시명(태국어 등)**·**연락처** 편집. |

### 3.3 홈 — 이달 접수 건수

- **기준 달:** 한국 시간(KST) 기준 달입니다. 상단 화살표로 이전·다음 달을 볼 수 있습니다.
- **건수 의미:** 그 달 **예약일**이 해당하는 접수 중, **취소·보류가 아니고**, 접수 **현장 인원 메모**에 본인 이름(한글 또는 등록된 태국어 표시명)이 맞게 적힌 건수입니다. **현장 일정**과 같은 기준입니다.
- 그래프에서 이름 옆에 **태국어 표시명**이 있으면 함께 표시됩니다.

### 3.4 일정(현장 일정)

1. **날짜**를 선택합니다 (처음에는 보통 **오늘(KST)**).
2. 그날 그룹에 속한 팀원을 기준으로, 메모에 붙은 **접수 건만** 행으로 펼쳐 보여 줍니다.
3. **배정 팀장·타업체 표기·희망 시간·차량번호** 등을 한눈에 확인합니다.
4. **집계·일자 명단 모드**인 그룹에서는, 그날 **명단에 없는** 팀원은 일이 있어도 “명단 밖” 느낌으로 표시될 수 있습니다(화면의 `onRoster` 등).

데이터가 바뀌면 서버에서 알림이 오면 **자동으로 다시 불러오는** 패턴이 적용될 수 있습니다(실시간 갱신).

### 3.5 일자 명단

1. **일자 명단** 메뉴 → **달력**에서 월을 바꿀 수 있습니다.
2. 날짜를 누르면 **그날의 편집 화면**으로 들어갑니다.
3. **그룹장**
   - 왼쪽 **멤버** 목록에서 사람을 선택한 뒤 **→** 를 누르면 오른쪽 **일할 멤버**로 옮깁니다.
   - 오른쪽에서 선택하고 **←** 를 누르면 빠집니다.
   - **저장** 시, 그룹이 **설정용 비밀번호**를 쓰도록 돼 있으면 비밀번호를 함께 입력해야 합니다.
4. **일반 멤버**는 보통 **조회만** 가능합니다.

**물음표(?) 아이콘**을 누르면 해당 화면의 짧은 도움말이 펼쳐집니다.

### 3.6 설정 (그룹장 전용)

- 상단에 **「설정」** 이 보이지 않으면, 관리자에게 **그룹장 지정**을 요청해야 합니다.
- **멤버 표시 이름:** 한글 이름 아래에 쓰는 **보조 이름(태국어 등)**. 각 행마다 **저장**을 눌러야 그 사람 것만 반영됩니다.
- **연락처:** 행의 **수정**으로 모달에서 바꿀 수 있습니다.
- 설명이 길면 제목 옆 **물음표**로 안내를 펼칩니다.

### 3.7 그룹장이 없을 때

- 관리자가 멤버는 넣었지만 **그룹장을 한 명도 지정하지 않은 경우**, 로그인은 되지만 **일자 명단 저장·설정**은 할 수 없습니다. 반드시 관리자가 **한 명을 그룹장**으로 지정해야 합니다.

## 5. 자주 묻는 질문 (FAQ)

**Q. 로그인이 안 돼요.**  
A. 관리자에게 **로그인 ID·비밀번호**를 다시 확인하세요. 그룹이 **중지**되었거나 ID 오타·대소문자(허용 규칙에 따라) 문제일 수 있습니다.

**Q. 일정에 제 이름이 안 나와요.**  
A. 해당 날짜 접수의 **현장 인원 메모**에 적힌 이름이, 시스템의 **한글 이름 또는 태국어 표시명**과 **완전 일치**하는지 확인하세요. **집계·일자 명단 모드**면 그날 **명단에 본인이 포함**되어 있는지도 확인하세요.

**Q. 홈 그래프가 0이에요.**  
A. 이번 달에 메모 매칭된 **취소/보류가 아닌** 접수가 없거나, 이름이 안 맞으면 0으로 나올 수 있습니다.

**Q. 설정 메뉴가 없어요.**  
A. **그룹장**으로 지정된 계정에서만 보입니다. 관리자에게 그룹장 지정을 요청하세요.

**Q. 명단 저장이 비밀번호를 요구해요.**  
A. 관리자가 **집계·일자 명단 모드**와 함께 **설정용 비밀번호**를 걸어 두었습니다. 그룹장에게 별도로 전달받은 **설정용 비밀번호**를 입력하세요. (공유 로그인 비밀번호와 다를 수 있습니다.)

## 6. 관련 경로·용어 (참고)

| 용어 | 의미 |
|------|------|
| 공유 로그인 ID | `TeamCrewGroup.loginId`, 크루 로그인에 사용 |
| 집계·일자 명단 모드 | `useDailyRosterOnly` — 명단에 올린 사람만 현장 후보 |
| 설정용 비밀번호 | 명단 저장 등 민감 조작 시 2차 확인 (`settingsPasswordHash`) |
| 그룹장 | `TeamCrewGroupMember.isGroupLeader` — 한 멤버만 지정하는 것을 권장 |

문서는 코드베이스의 동작을 기준으로 작성되었으며, 배포 환경에 따라 메뉴 이름이 약간 다를 수 있습니다. 최신 정책은 **팀원 관리** 화면의 안내 문구를 함께 확인하세요.
