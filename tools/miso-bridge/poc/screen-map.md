## 채팅 상세 (= 숨고 채팅방 + extract)

| UI 요소 | 설명 | selector/좌표 후보 | 비고 |
|---------|------|---------------------|------|
| 뒤로 | 채팅 목록 복귀 | clickable `[42,189][105,252]` | 상단 ← |
| **요청 요약 박스** | **탭 → 상세 정보** (숨고 고객요청 모달과 동급) | content-desc `6월 30일(화), 기한 만료, 이사청소/입주청소, ￦ 상담 후 결정` · bounds `[42,328][1038,534]` | **extract 1단계** |
| 헤더 전화 | | clickable `[933,168][1038,273]` | 고용 후 등 — PoC 추가 확인 |
| 고객명 | 마스킹 | text `이*화` | |
| 메시지 버블 | 대화 내용 | content-desc = 본문 | |
| 입력창 | send-message | hint `메시지를 입력해주세요` | |
| 전송 버튼 | | content-desc `send` | |

### extract 흐름 (숨고 `extract_request_modal` 대응)

```
대화하기 목록 → 행 탭 → 채팅 상세
  → 이름 아래 요약 박스 탭 (날짜 · ₩ 상담 후 결정 · 기한/서비스 뱃지)
  → **요청 정보 화면** UIAutomator scrape
  → 상단 ← 탭 → 채팅 상세 복귀
```

| 단계 | 숨고 | 미소 |
|------|------|------|
| 정보 진입 | 고객 요청 모달 | **헤더 요약 박스** 탭 |
| scrape | DOM | **text + content-desc** 전수 |
| 복귀 | 모달 닫기 | **상단 ←** |

자동 테스트: [probe_extract_request.py](./probe_extract_request.py) — **채팅 상세(이*화) 화면에서** 실행.

### 헤더 박스 content-desc → 필드 (초안)

| segment | 예 | extract 필드 |
|---------|-----|--------------|
| 1 | `6월 30일(화)` | serviceDate |
| 2 | `기한 만료` | rawStatusText → statusCode |
| 3 | `이사청소/입주청소` | serviceType |
| 4 | `￦ 상담 후 결정` | priceHint |

상세 화면 필드는 **2026-07-23 실측(이*화 건)** 기준 [04-data-schema.md](../docs/04-data-schema.md) · 아래 §요청 정보 화면.

## 요청 정보 화면 (헤더 박스 탭 후 = 숨고 고객요청 모달)

**진입:** 채팅 상세 → 요약 박스 탭  
**제목(GNB):** `이사청소/입주청소`  
**복귀:** 상단 ← → 채팅 상세

### 상단

| UI 라벨 (앱 원문) | 예시 값 | extract 필드 |
|-------------------|---------|--------------|
| (뱃지) | `기한 만료` | `statusLabel` → `EXPIRED` |
| (대표 일정) | `6월 30일(화)` | `scheduledAt` / `serviceDate` |
| 메모 | `메모를 입력해주세요` | `partnerMemo` (입력 가능) |

### 견적 정보

| UI 라벨 | 예시 값 | extract 필드 |
|---------|---------|--------------|
| 견적 금액 | `상담 후 결정` | `quoteAmount` (문자열 또는 null) |
| 견적 제출 일시 | `6월 22일(월) 오후 4시 35분` (+ `자동` 뱃지) | `quoteSubmittedAt` |

### 주문 정보

| UI 라벨 | 예시 값 | extract 필드 |
|---------|---------|--------------|
| 평 수 | `24 (전용)` | `areaPyung` |
| 서비스 타입 | `이사청소` | `serviceTypeDetail` |
| 방 개수 | `거실 + 방 2개` | `roomLayout` |
| 거주지 종류 | (스크롤 하단) | `residenceType` |
| (추가 행) | 주소·옵션 등 | PoC 스크롤 후 수집 |

**scrape 방식:** 화면 `text` 노드를 위→아래 순회 — **라벨 TextView 다음 값 TextView** 쌍으로 파싱 (`probe_extract_request.py`).
